---
1. QLY(Quest-ly): 누구나 요청하고 누구나 도와주는 생활 퀘스트 플랫폼
2. 목차.
3. 프로젝트 팀원 소개.
- 김희상: 프로젝트 총괄 기획, 전체 플랫폼 기능 흐름 설계 및 기술 방향 제시, DB 구조 논의 및 일정 조율, 코인 결제 기반 보상 로직, 설계 및 연동 관리
- 김성진: 프론트엔드 & 디자인 총괄, 전체 페이지 UI/UX 설계 및 구현. 관리자 페이지 기능 구현
- 박윤재: 백엔드 개발 & 구조화, 사용자 페이지 기능 구현 담당, 1:1 문의 기능 및 답변 프로세스 흐름 설계
- 정종일: 백엔드 개발 & 구조화, DB 연동 기반의 퀘스트, 등록/조회 기능 구현, 메인 로직 중심의 백엔드, 구조화 및 서비스 간 API 통합
- 조형진: 프론트엔드 & 디자인, 로그인 페이지 및 코인 환급 페이지 퍼블리싱
4. Project Background: 생활 퀘스트 플랫폼 QLY, 왜 필요한가요?
- 1인 가구의 급증과 고령화, 초개인화된 생활 구조의 확산은 가족·이웃 기반의 돌봄을 약화시키고, 병원 동행·가사 지원·정서적 교류 같은 작지만 반복적인 일상 도움에 대한 사회적 수요를 빠르게 증가시키고 있습니다.
5. QLY의 해결방안 & 방향성
- “생활 속 도움 연결”: 일상 속 가사, 병원 동행, 정서 교류 등 혼자 해결하기 어려운 일상을 요청하고 해결
- "서로 돕는 참여 구조": 누구나 의뢰자이자 해결사. 함께 만들어가는 상생 네트워크
- "간편한 연결 플랫폼": 웹에서 간편하게 도움이 필요한 사람과 해결사를 신속하게 연결
- 결론: 도움을 요청하는 의뢰자, 도움을 주는 해결자가 서로 되어주는 상생형 생활 퀘스트 플랫폼입니다. 일상의 작은 문제들, 이제 함께 연결하고 함께 해결합니다.
6. QLY의 페르소나: 의뢰자 / 해결사 / 관리자\
- 의뢰자: "일상 생활속의 도움을 요청하는 사용자". 퀘스트 요청 등록, 토큰 결제 & 등록, 퀘스트 조회 및 상태 확인, 퀘스트의 해결사(Pro) 선정 및 퀘스트 진행.
- 해결사: "퀘스트를 수행하여 보상을 받는 사용자". 퀘스트 목록 열람, 원하는 퀘스트 신청, 의뢰자와 매칭, 퀘스트 수행, 완료 후 보상 수령
- 관리자: "플랫폼 전반 운영 및 관리". 회원 관리, 퀘스트 관리, 코인 관리, 1:1 문의 응대, 운영 메모 및 공지사항 게시
7. QLY의 키워드: 사용자(의뢰자 및 해결사), 퀘스트(Quests), 보상(Reward), 서비스(Service)
- 사용자(의뢰자 및 해결사): 1:1 매칭, 마이페이지(활동관리)
- 퀘스트(Quests): 퀘스트 등록, 퀘스트 신청
- 보상(Reward): 코인 결제/환급(PortOne API 활용)
- 서비스(Service): 고객센터 1:1 문의, 카카오Map API 활용 지역 카테고리 필터
8. QLY의 주요 서비스
- 1. 의뢰자(Client) ↔ 수행자(Pro) 간 매칭
- 2. 코인 기반 보상 시스템
- 3. 위치 기반 필터링 & 카테고리 검색
- 4. 퀘스트 진행 관리 & 체크리스트
- 5. 안전한 거래 보장 /1:1 문의
- 6. 프로젝트 확장성 & 관리자 운영 편의성
9. QLY의 서비스 흐름도 7단계
- 1. 회원가입 & 로그인: 의뢰자(Client) /  해결사(Pro) 사용자 역할 선택
- 2. 의뢰자 퀘스트 등록: 의뢰자(Client)가 요청할 퀘스트 업로드 (요청 항목, 장소, 시간, 상금 등 상세 정보 입력)
- 3. 해결사 퀘스트 신청: 해결사가 원하는 퀘스트 확인 후 신청
- 4. 의뢰자 - 해결사 매칭: 조건이 맞는 해결사와 1:1 매칭되어 퀘스트 수행 준비 완료
- 5. 퀘스트 수행: 해결사가 지정된 장소 및 시간에 퀘스트 수행
- 6. 퀘스트 상태 확인: 의뢰자와 해결사는 마이페이지에서 현재 진행 중인 퀘스트 실시간 확인 가능 (상태: 대기중 → 진행중 → 완료)
- 7. 결과 확인 및 보상: 의뢰자가 퀘스트의 각 체크리스트 완료 확인 후 해결사에게 보상(코인) 지급
10. QLY 시스템 구조 흐름도(각 페르소나 기준):
- 의뢰자(Client) -> 퀘스트 등록 -> 퀘스트 매칭 -> 퀘스트 완료 체크 -> 해결사에게 보상 지급
- 해결사(Pro) -> 퀘스트 조회 -> 퀘스트 신청 -> 퀘스트 매칭 -> 퀘스트 진행 및 완료 -> 의뢰자에게 보상 수령
- 관리자(Admin) -> 관리자 페이지 접근 -> 전체 통계 대시보드 -> 유저 및 코인, 퀘스트 관리

🎤 QLY 발표 대본 (김희상 파트)
1. QLY 소개
안녕하세요. 저희 프로젝트의 이름은 Quest-ly, 줄여서 QLY 입니다.
일상을 퀘스트처럼 생각하자는 의미가 담겨있습니다. 
“누구나 요청하고, 누구나 도와주는 생활 퀘스트 플랫폼”이라는 슬로건 아래, 
상생과 연결의 가치를 담은 웹 서비스를 기획했습니다.

2. 발표 목차 안내
오늘 발표는 다음과 같은 순서로 진행하겠습니다. 
먼저 프로젝트 개요와 기획 의도, 그 다음으로 서비스 구조, 사용자 흐름, 기술 구성 등 
서비스의 전반적인 구성과 특징을 소개해 드리겠습니다.

3. 팀원 소개
저희는 총 5명으로 구성된 팀입니다.

저, 김희상은 이 QLY 프로젝트의 팀장으로써 
총괄 기획과 함께 전체 서비스 흐름 설계, 기술 방향 제시, DB 구조 논의, 
그리고 코인 결제 기반의 보상 로직 설계 및 연동을 맡았습니다.

김성진 팀원은 프론트엔드 전체 및 UI/UX 디자인을 총괄했고, 
또한 관리자 기능 페이지 개발도 담당했습니다.

박윤재 팀원은 백엔드 구조화 및 사용자 페이지 기능을 중점적으로 맡았고, 
1:1 문의 처리 로직도 설계했습니다.

정종일 팀원은 DB 기반의 퀘스트 등록/조회 기능과 
API 통합을 포함한 핵심 로직 설계를 담당했습니다.

조형진 팀원은 프론트엔드 퍼블리싱을 맡아 
로그인 페이지 및 코인 환급 페이지를 구현했습니다.

4. 프로젝트 배경
왜 QLY가 필요했을까요?

최근 1인 가구의 급증, 고령화, 초개인화된 생활 구조는 전통적인 돌봄 구조를 약화시키고 있습니다. 
이로 인해 병원 동행, 간단한 가사지원, 정서적 교류처럼 일상에서 반복되는 작은 도움 요청들이 
사회적 수요로 빠르게 증가하고 있습니다.

5. QLY의 해결방안 & 방향성
저희는 세 가지 키워드로 문제를 해결하고자 했습니다.

- 1. 생활 속 도움 연결: 일상 속 어려움을 퀘스트로 등록하고, 해결사와 매칭하여 직접 수행되는 구조입니다.
- 2. 서로 돕는 구조: 누구든 의뢰자이자 해결사가 될 수 있어 상호 연결의 참여형 생태계를 구축했습니다.
- 3. 간편한 연결 플랫폼: 웹 기반으로 쉽게 매칭부터 완료까지 진행되도록 사용성 중심의 설계를 지향했습니다.

결론적으로, 서로 도움을 주고받는 생활 퀘스트 플랫폼, 그것이 QLY입니다.

6. QLY 페르소나
플랫폼은 총 3가지 사용자를 중심으로 설계했습니다.

의뢰자(Client): 퀘스트를 등록하고, 해결사를 선정한 후 진행 상황을 확인하고 평가합니다.
해결사(Pro): 등록된 퀘스트를 확인하고 신청하며, 매칭된 퀘스트를 수행한 뒤 보상을 받습니다.
관리자(Admin): 전체 회원, 퀘스트, 코인, 문의사항을 관리하고 운영 공지 등을 게시합니다.

🎯 7. 핵심 키워드 4가지 소개
QLY 서비스는 다음 네 가지 핵심 키워드를 중심으로 기능을 설계했습니다.

첫 번째는 사용자(User)입니다. 의뢰자와 해결사 각각의 마이페이지를 통해 개인 퀘스트 현황을 확인할 수 있고, 1:1 매칭 기능을 통해 정확한 사용자 연결이 가능합니다.

두 번째는 퀘스트(Quests)입니다. 의뢰자가 직접 퀘스트를 등록하고, 해결사가 이를 신청하는 구조로, 자율성과 선택 기반의 매칭 흐름을 유연하게 구성했습니다.

세 번째는 보상(Reward)입니다. PortOne API 기반의 결제/환급 로직을 통해, 의뢰자가 코인을 충전하고, 퀘스트 완료 후 해결사가 이를 수령할 수 있도록 구현했습니다.

마지막으로 서비스(Service)입니다. 카카오맵 기반의 위치 필터링과 1:1 문의 기능을 통해 사용자 편의를 높였고, 다양한 기능들이 유기적으로 연결되어 있는 것이 QLY의 큰 특징입니다.

⚙️ 8. 주요 서비스 정리
저희가 구현한 주요 서비스는 다음과 같습니다.

의뢰자와 해결사 간 1:1 매칭 시스템

코인 기반 보상 시스템을 통한 동기 부여

카카오맵 API 활용으로 위치 기반 퀘스트 탐색

체크리스트 기반의 퀘스트 진행 관리

그리고 1:1 문의 처리 시스템과 관리자 기능 강화 및 서비스 확장 가능성까지 고려된 구조입니다.

모든 기능은 실사용을 고려해 실제로 작동 가능한 흐름으로 설계했습니다.

🔄 9. 프로젝트를 관통하는 메인 서비스 로직 FLOW
QLY 서비스는 총 7단계의 흐름을 가집니다.

사용자는 회원가입 및 로그인 시에 의뢰자 또는 해결사 역할을 선택하게 됩니다.

의뢰자는 퀘스트를 등록하면서 요청 항목, 위치, 시간, 보상 등을 입력하고

해결사는 등록된 퀘스트 목록을 열람한 뒤 원하는 퀘스트를 신청합니다.

신청이 완료되면 조건 기반으로 의뢰자와 해결사가 자동 매칭됩니다.

매칭된 해결사는 지정된 시간과 장소에서 퀘스트를 수행하게 되고

의뢰자와 해결사 모두 마이페이지에서 퀘스트 진행 상태를 실시간으로 확인할 수 있습니다.

마지막으로 의뢰자가 퀘스트의 체크리스트 완료 여부를 검토한 후, 조건을 만족했을 경우 보상 지급 버튼이 활성화되어 해결사에게 코인이 지급됩니다.

👥 10. 시스템 구조 흐름도
QLY 플랫폼은 사용자 유형별로 역할과 흐름이 명확히 구분되어 있습니다.

- 의뢰자(Client)는 퀘스트를 등록하고 → 매칭 결과를 확인 → 퀘스트 진행 완료 여부를 체크하고 → 최종적으로 보상을 지급합니다.

- 해결사(Pro)는 퀘스트를 열람하고 → 원하는 퀘스트를 신청 → 매칭된 퀘스트를 수행하고 → 완료 후 보상을 수령합니다.

- 관리자(Admin)는 관리자 페이지를 통해 → 회원과 퀘스트 현황을 관리하고 → 통계 대시보드, 코인 흐름, 문의사항 응대를 총괄하며 플랫폼의 운영을 지원합니다.

이 파트를 통해 QLY가 단순한 연결 플랫폼이 아니라, 사용자 흐름, 역할 분담, 기술적 신뢰도까지 고민한 “실제 작동하는 서비스”라는 점을 전달하고자 했습니다.

이상으로 QLY의 기본 방향성과 서비스 흐름을 소개해 드렸고, 다음으로는 시스템 구조와 주요 기능에 대해 이어서 설명드리겠습니다.
