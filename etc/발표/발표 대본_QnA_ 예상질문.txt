🧭 1. 메인페이지
❓Q1. 메인페이지에서 가장 중점적으로 고려한 UX 요소는 무엇인가요?
🗣A. 사용자가 처음 접속했을 때 "내가 지금 뭘 할 수 있지?"를 바로 알 수 있도록 퀘스트 등록/확인 등의 핵심 기능을 상단에 배치하고, 카테고리 필터와 추천 퀘스트 섹션으로 직관적인 사용자 여정을 설계했습니다.

❓Q2. 메인페이지에서 실시간 정보는 어떻게 반영되나요?
🗣A. 퀘스트 등록 시간 기준으로 최신 등록순 정렬과 실시간 상태 표시 기능을 넣어 사용자가 현재 진행 중인 퀘스트나 마감 임박 퀘스트를 빠르게 확인할 수 있도록 구성했습니다.

🎯 2. 퀘스트: 등록 → 신청 → 진행 → 완료
❓Q1. 퀘스트 등록 시 사용자 편의성을 위해 어떤 기능을 구현했나요?
🗣A. 사진 업로드 미리보기, 카카오맵 기반 위치 설정, 멀티 필드 입력 저장 등을 통해 복잡한 입력도 직관적이고 간단하게 완료할 수 있도록 설계했습니다.

❓Q2. 퀘스트 진행 및 완료 과정에서 가장 중점적으로 고려한 로직은 무엇인가요?
🗣A. 퀘스트별 진행 단계에 따른 상태 관리와 체크리스트 기반 완료 검증 로직을 적용해 정확한 퀘스트 흐름 관리와 보상 지급 조건을 자동으로 제어할 수 있도록 했습니다.

💰 3. 코인 충전 및 환급
❓Q1. 코인 충전과 환급은 어떤 방식으로 처리되나요?
🗣A. PortOne API를 연동하여 안정적인 결제 및 환급 기능을 구축했고, 코인 히스토리는 별도의 AOP 로직을 통해 자동 이력화되어 거래의 추적성과 신뢰성을 보장합니다.

❓Q2. 코인 관련 예외 상황(예: 중복 결제, 실패 등)은 어떻게 처리되나요?
🗣A. 결제 실패 시에는 트랜잭션 단위에서 예외 처리를 수행하고, 사용자에게 오류 메시지 및 재시도 안내를 제공합니다. 중복 결제 방지를 위한 프론트/백엔드 이중 검증 로직도 구현했습니다.

📂 4. 마이페이지 / 관리자페이지
❓Q1. 마이페이지에서는 어떤 정보와 기능이 제공되나요?
🗣A. 사용자의 퀘스트 활동 내역, 진행 상태, 누적 보상 정보 등이 한눈에 보이며 진행 중인 퀘스트 실시간 확인, 평가 기록, 1:1 문의 내역 확인까지 가능하게 구성했습니다.

❓Q2. 관리자페이지는 어떤 기능과 목적을 가지고 있나요?
🗣A. 전체 유저 및 퀘스트 데이터 관리, 코인 거래 현황 확인, 문의사항 응답, 운영 메모 작성 및 공지 게시 등의 기능이 있으며, 운영의 효율성과 유지보수를 고려한 구조로 설계했습니다.

💬 5. 문의사항 페이지
❓Q1. 1:1 문의 기능은 어떻게 동작하나요?
🗣A. 사용자와 관리자가 주고받는 비동기식 채팅 형식의 문의처리 시스템을 구현했으며, 관리자는 별도의 알림과 목록 필터 기능을 통해 효율적으로 대응할 수 있습니다.

❓Q2. 문의사항 페이지와 유사 서비스 대비 차별점은 무엇인가요?
🗣A. QLY는 단순한 문의 등록이 아니라, 문의 내역 확인 + 관리자 답변 확인 + 마이페이지 연동까지 한 번에 처리할 수 있는 구조로 사용자의 접근성과 피드백 처리 속도를 높였습니다.
